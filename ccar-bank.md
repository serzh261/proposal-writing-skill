# CCAR Bank — [Firm] Past Performance

CCAR = Client / Challenge / Action / Result. The standard format for past performance citations in government proposals. Each entry must survive a reference call — never invent specifics. Verify-flags `[VERIFY: ...]` mark items pulled from project history that [Principal] must confirm or correct before submission.

## How to use this bank

For any proposal, pick 3-5 CCARs most relevant to the bid. Lead with the most aligned, not the most impressive. Use the tags below each entry to filter.

When drafting past performance section, write a 2-3 sentence narrative for each entry (don't paste the full structured fields into the proposal — those are working notes). Add a "Relevance to [Buyer]:" callout linking each entry to the specific evaluation criterion it supports.

## Maintenance Notes

After each win:
1. Convert the pending CCAR to a complete one with real Client / Challenge / Action / Result content
2. Add reference contact (name, title, email, phone) with verified willingness to serve as reference
3. Note CPARS rating if federal and rating issued
4. Add tags reflecting actual scope delivered

After each year:
1. Re-verify reference contacts (people leave jobs)
2. Refresh tags as [Firm]'s positioning evolves
3. Retire CCARs older than 5 years unless they're uniquely strong

## Selecting CCARs for a bid

Match by tag first, then by scale and recency:

- Federal civilian bid → CCAR 1 ([Federal Intermediary]) leads, CCAR 2 (AHP) supports
- State/regional non-federal bid → CCAR 3 ($[REDACTED]M pipeline) leads, others support
- Higher-ed multi-campus bid → CCAR 1 leads (multi-jurisdictional pattern matches), CCAR 7 (UM) once available
- [Metro] regional bid → CCAR 6 ([Regional Council]) once available, CCAR 1 supports
- Emergency services / public safety bid → CCAR 6 once available, CCAR 1 for federal funding context

If fewer than 3 strong matches, supplement with strong adjacent CCARs and acknowledge the adjacency in the Relevance callout.
