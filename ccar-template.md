## How to write a defensible CCAR

1. **Use specific numbers only when verifiable.** "$[REDACTED]M federal program" only works if you can document $[REDACTED]M. If unsure, flag `[VERIFY: $[REDACTED]M figure — confirm with [Federal Intermediary] records or [Principal]'s prior contracts]`.

2. **Active voice, concrete actions.** "Built and operated the analytics environment" not "Was responsible for analytics oversight."

3. **Connect to evaluation criteria when used in proposal.** The CCAR bank entry is for tracking. When you pull it into a proposal, add a "Relevance to [Buyer]:" callout connecting this project to a specific evaluation factor.

4. **Verify reference contacts within 30 days of any proposal submission.** People leave jobs. A reference contact at an organization the person no longer works at fails on the first call attempt and damages the proposal.

5. **Don't fold multiple projects into one CCAR.** If [Principal] did three engagements with the same client, they're three CCARs. Each one is a separate citation.

6. **Don't claim CPARS ratings you don't have.** Only federal contracts with completed performance periods generate CPARS. State/commercial projects don't.

7. **Update annually.** Re-verify references, refresh tags, retire CCARs older than 5 years unless uniquely strong.

## Status flag conventions

- `[VERIFY: specific item]` — uncertain detail that [Principal] must confirm before use
- `[PENDING AWARD]` — proposal submitted but not yet won; cannot be used as past performance yet
- `[PENDING DELIVERY]` — contract won but work not yet completed; cannot cite full results
- `[ANONYMIZED — name available on request]` — client name confidential
- `[RETIRED]` — no longer used; kept for historical record only
